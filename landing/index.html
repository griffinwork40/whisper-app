Whisper App

Transcribes your speech on-device using mlx-whisper and Apple's Neural Engine — no internet connection, no background service, no Dock icon. Unlike Apple Dictation, nothing reaches Apple's servers and the model is open-source, auditable, and runs entirely on your chip.

v0.1.0 — source install only. Not yet code-signed or notarized. Apple Silicon required. Tested on macOS Sequoia.
Default hotkey ⌥ ⌘ hold to talk

Install from source

The app is not yet code-signed — source install only, no .dmg or Homebrew formula yet. You need Node.js, Python 3 with pip, and Homebrew's ffmpeg.

One-time setup (~5 min) 
# Python transcription backend
pip3 install mlx-whisper

# ffmpeg — path is currently hardcoded to /opt/homebrew/bin/ffmpeg
brew install ffmpeg

# Pre-cache the default model (whisper-turbo, ~810 MB)
python3 -c "from huggingface_hub import snapshot_download; snapshot_download('mlx-community/whisper-turbo')"

Note on ffmpeg: The path /opt/homebrew/bin/ffmpeg is hardcoded in src/recorder.ts. If your Homebrew prefix differs (Intel Macs use /usr/local/bin), edit that file and rebuild before running.

Clone, build, and run (~2 min) 
git clone https://github.com/griffinwork40/whisper-app.git
cd whisper-app
npm install
npm run build
npm start

The tray icon appears in your menu bar immediately. Hold ⌥ ⌘ to record, release to transcribe. On first use, macOS will prompt for Microphone access (and Accessibility permission if you select autotype output mode).

From hotkey to clipboard

Whisper App runs entirely inside a single Electron main process — there is no renderer, no browser window, no persistent background daemon. When you press the hotkey, ffmpeg begins capturing raw PCM audio from your default input device. When you release it, the WAV file is handed to a Python shim that calls mlx_whisper.transcribe() — the transcription runs on-chip via the Neural Engine or GPU — and the result is written to your clipboard (or typed directly into the focused application via AppleScript). The temp file is deleted immediately. Nothing leaves your machine at any point.

  1. 01 Hotkey pressed → ffmpeg starts capturing 16 kHz mono PCM16 from the system default input device.
  2. 02 Hotkey released → ffmpeg flushes and closes the WAV at /tmp/whisper-<uuid>.wav.
  3. 03 scripts/transcribe.py is spawned with the WAV path and model id. It calls mlx_whisper.transcribe() and prints the transcript to stdout.
  4. 04 Transcript is written to the clipboard and/or typed into the focused app using AppleScript System Events.
  5. 05 Temp WAV is deleted. No network calls at any point.

Under the hood

Transcription engine

Uses mlx-whisper — Apple's MLX port of OpenAI Whisper. The default model is mlx-community/whisper-turbo (~810 MB, cached to ~/.cache/huggingface/hub/). Larger models (whisper-large-v3-turbo, whisper-large-v3-mlx) are selectable from the tray menu. All model code is open-source and auditable.

Hardware acceleration

MLX runs inference on the Neural Engine and GPU through Apple's Accelerate / Metal stack. No CUDA, no ROCm. On an M4 Pro, a 10-second recording with whisper-turbo transcribes in under 5 seconds from the moment the hotkey is released.

Silence gate

A −45 dBFS silence threshold is applied during recording. Frames below that level are still captured — the gate detects speech onset rather than trimming the WAV — so the model receives the full utterance including natural leading silence.

Hotkey modes

Hold to talk uses uiohook-napi to observe raw key events, which works for modifier-only chords (e.g. ⌥⌘). Tap to toggle uses Electron's globalShortcut and works for any accelerator string.

Output modes

Three modes selectable from the tray: clipboard (writes the transcript string via Electron's clipboard API), autotype (plays back keystrokes into the focused application via AppleScript System Events keystroke), and both. Autotype requires Accessibility permission.

Process model

Main-process-only Electron. No BrowserWindow, no renderer, no webContents. The tray icon and context menu are the entire UI surface. App state cycles through four values: idle, recording, processing, error.

Architecture

Everything runs inside a single operating-system process. The Python transcription script is a short-lived child spawned per recording, not a persistent service.
